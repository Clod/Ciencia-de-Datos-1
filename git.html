La Arquitectura Local

Para entender Git, debemos entender que nuestro código vive en tres "estados" o lugares distintos:

Working Directory

Tus archivos físicos. Es el editor donde escribes tu código.

Staging Area

La zona de preparación. Aquí seleccionas qué cambios vas a guardar.

Local Repo

El historial oficial. Donde se guardan las versiones permanentes (commits).

El Problema sin Git

Imagina que estás escribiendo un libro. Sin control de versiones:

libro_final.txt
libro_final_v2.txt
libro_final_final_este_si.txt

Caos total: ¿Cuál es la versión correcta? ¿Dónde está cada cambio?

17 / 23

La Solución: Instantáneas (Snapshots)

Con Git, solo tienes un archivo. Git guarda "fotos" de tu proyecto en diferentes momentos.

V1 - Primer Capítulo
10/01/2025 - "Versión inicial"
V2 - 添加 Segundo
15/01/2025 - "Agregados 3 capítulos"
V3 - Revisión Final
20/01/2025 - "Correcciones finales"

Puedes regresar a cualquier versión anterior en segundos.

Configuración Inicial

Antes de empezar, identifica tu usuario ante Git:

$ git config --global user.name "Tu Nombre"
$ git config --global user.email "tuemail@ejemplo.com"

El flag --global aplica esta configuración a TODOS tus proyectos.

Comandos Esenciales

git init

Crea un nuevo repositorio en la carpeta actual.

git status

Muestra el estado actual de tus archivos.

git add .

Añade TODOS los cambios al Staging.

git commit -m "msg"

Guarda los cambios con un mensaje descriptivo.
